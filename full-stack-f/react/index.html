                  # 生命周期（旧）

                  react生命周期总共有四条线，一个是组件初次挂载的时候，父组件render触发组件更新的时候，组件setState的时候和组件forceUpdate的时候，卸载需要手动卸载。

                  • constructor 构造函数

                  • componentWillMount 组件将要挂载

                  • componentDidMount 组件挂载完毕

                  • componentWillReceiveProps 组件将要接收props的时候

                  • shouldComponentUpdate 控制组件更新的钩子需要返回一个布尔值，默认返回true

                  • componentWillUpdate 组件将要更新的钩子

                  • componentDidUpdate 组件更新完毕的钩子

                  • render 渲染页面

                  • componentWillUnmount 卸载组件

                  组件挂载过程:

                  • constructor 构造函数

                  • componentWillMount 组件将要挂载

                  • componentDidMount 组件挂载完毕

                  • render 渲染页面

                  父组件render触发组件更新过程:

                  • componentWillReceiveProps 组件将要接收props的时候

                  • shouldComponentUpdate 控制组件更新的钩子需要返回一个布尔值，默认返回true

                  • componentWillUpdate 组件将要更新的钩子

                  • render 渲染页面

                  • componentDidUpdate 组件更新完毕的钩子

                  调用setState更新过程：

                  • shouldComponentUpdate 控制组件更新的钩子需要返回一个布尔值，默认返回true

                  • componentWillUpdate 组件将要更新的钩子

                  • render 渲染页面

                  • componentDidUpdate 组件更新完毕的钩子

                  调用forceUpdate更新过程：

                  • componentWillUpdate 组件将要更新的钩子

                  • render 渲染页面

                  • componentDidUpdate 组件更新完毕的钩子

                  # 生命周期（新）

                  改名的生命周期函数并且已经不出现在官网更新的生命周期示例图中：

                  • UNSAFE_componentWillMount 组件将要挂载

                  • UNSAFE_componentWillUpdate 组件将要更新的钩子

                  • UNSAFE_componentWillReceiveProps 组件将要接收props的时候

                  虽然没有直说，但是未来可能会废弃的三个钩子，目前可以用但是必须加上UNSAFE前缀。

                  因为react为了给以后要做的异步渲染铺路，加前缀的这三个钩子容易被误用，在未来版本可能会出现bug，所以修改其命名方式让人用起来不是很方便减少误用可能性，其实这三个钩子在项目中应用场景也是非常少的，基本做完整个项目不用也很正常，因为它们也不是什么核心钩子。

                  新的生命周期：

                  • constructor 构造函数

                  • getDerivedStateFromProps 从props中获取state并且映射到state上面

                  • componentDidMount 组件挂载完毕

                  • shouldComponentUpdate 控制组件更新的钩子需要返回一个布尔值，默认返回true

                  • getSnapshotBeforeUpdate 更新前获取快照

                  • componentDidUpdate 组件更新完毕的钩子

                  • render 渲染页面

                  • componentWillUnmount 卸载组件

                  getDerivedStateFromProps和getSnapshotBeforeUpdate是新出的钩子，使用场景非常罕见。

                  getDerivedStateFromProps使用时必须加上static变成静态方法并且需要返回一个状态对象或者null，返回的状态对象将会映射到当前组件的状态中，因为这个生命周期函数横跨组件挂载和组件更新之前（即render和shouldComponentUpdate之前），所以当你返回状态对象后当前组件的state将不可修改，只能使用返回的对象作为当前组件的state。

                  getSnapshotBeforeUpdate处于render和componentDidUpdate之间，使用时必须返回一个快照或者null，它的返回值将传递给componentDidUpdat的第三个参数。

                  组件挂载过程：

                  • constructor 构造函数

                  • getDerivedStateFromProps 从props中获取state并且映射到state上面

                  • render 渲染页面

                  • componentDidMount 组件挂载完毕

                  父组件render触发组件更新过程（setState也是这个流程）：

                  • getDerivedStateFromProps 从props中获取state并且映射到state上面

                  • shouldComponentUpdate 控制组件更新的钩子需要返回一个布尔值，默认返回true

                  • getSnapshotBeforeUpdate 更新前获取快照

                  • componentDidUpdate 组件更新完毕的钩子

                  forceUpdate执行过程：

                  • getDerivedStateFromProps 从props中获取state并且映射到state上面

                  • getSnapshotBeforeUpdate 更新前获取快照

                  • componentDidUpdate 组件更新完毕的钩子

                  # setState

                  setState是改变state状态里面的值并且调用render函数更新UI，它接收两个参数，第一个是一个对象/回调函数，第二个是回调函数。

                  setState在合成事件（钩子函数和绑定事件）中的更新可能是“异步”的，在原生事件和setTimeout中是同步执行。但是所谓的异步并不是代表react在内部使用异步函数实现的setState，而是因为react为了兼容性把原生事件都封装了，我们在调用比如onClick的时候实际上没有直接调用原生的onclick事件，所以看起来像是异步是因为react内部实现更新状态是在合成事件之后才执行。想获取改变后的值可以在第二个参数回调函数中获取。

                  setState还有一个特点就是更新状态是批量合并，相同的值会取最后一个覆盖不会因为你调用多次就执行多次。

                  # 受控组件与非受控组件

                  • 受控组件 所有输入类或表单组件的状态即时更新到当前实例的state中，用的时候直接从state里面取

                  • 非受控组件 现用现取，用的时候直接从表单组件的value中取值，无需向state里面保存

                  # Fragment

                  有时候div嵌套层级过深我们不想要组件外层包裹的div节点就可以使用Fragment替换它，在react渲染的时候就会把它丢掉，主要是为了骗过react的语法检测，也可以直接使用<></>代替，不过在需要写key值的时候还是要用Fragment，空标签不支持写属性。

                  # Hooks

                  hook可以让你在函数式组件里面模拟生命周期函数的操作，

                  • useState 函数组件可以使用的响应式数据，返回两个值，一个是你传入的值，一个是set方法

                  • useEffect 模拟生命周期钩子的api，第一个参数是一个函数，第二个参数是数组

                  • useRef 创建ref

                  
function App() {
  /* 
    hooks userState因为react内部做了处理，虽然每次set改变状态后都会重新调用函数渲染页面，但是初始化值的时候react就已经把值保存了下 来，下次在调用不会执行初始化方法
  */
  const [name,setName] = React.useState('zs')
  // 创建ref对象，使用跟类式组件一样
  const myRef = React.useRef()
  // 不传第二个参数表示检测所有人，传了数组就首次加载完执行一次（componentDidMount），然后你每次变更监测的状态值发生改变也执行，传空数组表示谁也不监测
  React.useEffect(()=> {
    console.log('hi')
    // 返回的函数相当于componentWillUnmount，可以在里面处理一些卸载页面时的逻辑，比如清除定时器
    return ()=> {

    }
  },[name])
  function changeName() {
    // 赋值有两种写法
    // setName('ls')
    setName(()=> 'ls')
  }
  return (
    <div>
      <input ref={myRef} />
      <p>{name}</p>
      <button onClick={changeName}>改名</button>
    </div>
  )
}

                  # Context

                  在我们需要组件之间传递一些参数时，由于父子组件下面还有孙子组件等等，并且中间的组件并不需要这个值的时候想让孙子组件直接拿到需要的值就要用到Context。

                  
const { Provider,Consumer } = React.createContext

export default class A extends React.Component {
  state = {
    name: 'zs'
  }

  render() {
    return (
      <div>
        <Provider value={{ name:this.state.name }}>
          <B />
        </Provider>
      </div>
    )
  }
}

class B extends React.Component {
  render() {
    return (
      <div>
        <C />
      </div>
    )
  }
}

class C extends React.Component {
  // 声明接收
  static contextType = MyContext
  render() {
    return (
      <div>
        { this.context.name }
      </div>
    )
  }
}

// 上面的方式只能是类式组件使用，函数式需要再引用一个组件Consumer，组件内部写一个回调函数，参数为context的值

function D() {
  return (
    <div>
      <Consumer>
        {
          value=> {
            return <span>{ value.name }</span>
          }
        }
      </Consumer>
    </div>
  )
}

                  # 在react中模拟插槽

                  我们都知道在react组件标签内写入的内容在props.children能拿到，但是组件标签内部套组件呢？看到这里熟悉vue的伙伴应该已经闻到了slot的气息，没错，renderProps的方式和slot很相似，下面让我们看看具体用法：

                  
export default class A extends React.Component {
  render() {
    return (
      <div>
        <B render={name=> <C name={ name } />} />
      </div>
    )
  }
}

class B extends React.Component {
  state = {
    name: 'zs'
  }

  render() {
    return (
      <div>
        { this.props.render(this.state.name) }
      </div>
    )
  }
}

class C extends React.Component {
  render() {
    return (
      <div>
        { this.props.name }
      </div>
    )
  }
}

                  在这个案例里面组件标签的render属性返回一个组件，那么就代表我们可以使用this.props.render方法来插入任何我们想用的组件。 PS:需要注意的是render并不是固定写法，这个属性名写什么都可以，不过建议写render语义化比较好
